    Zip and Unzip folders in Python!

    Perfect for automating file compression and extraction tasks. 

                    
import os
import zipfile

def zip_folder(ip_folder, op_folder):
    with zipfile.ZipFile(op_folder, 'w') as zipf:
        for foldername, subfolders, filenames in os.walk("C:\\Books"):
            for fname in filenames:
                filepath = os.path.join(foldername, fname) 
                zipf.write(filepath,fname)
ip_folder = "C:\\Books" 
op_folder = "c:\\0Madhura\\Books.zip"

# Zip files
zip_folder(ip_folder, op_folder) 

# UnZip files
def unzip_folder(zipfolder, finalfolder):
    with zipfile.ZipFile(zipfolder, 'r') as zip_ref:
        zip_ref.extractall(finalfolder)
unzip_folder("c:\\0Madhura\\Books.zip", "c:\\0Madhura\\Books")

    CV Video Player - Once and For All

    A Python video player package made for computer vision research

    When developing computer vision algorithms, the journey from concept to working implementation often involves countless iterations of watching, analyzing, and debugging video frames.
    The same bolierplate code for video visualization and debugging needs to be written repeatedly. To solve this problem, the author of the article has created CVVideoPlayer package, a Python-based open-source video player package, specifically designed for computer vision practitioners.

    Debugging video-related algorithms is hard

    If you’ve ever developed an algorithm for video analysis, you’ve probably written some version of the following code to help you visualize and debug it:

     
import cv2
cap = cv2.VideoCapture("video_path")
ret = True
while ret:
  ret, frame = cap.read()
  algo_output = some_video_analsys_algorithm(frame)
  frame_to_display = visualizer(frame, algo_output)
  cv2.imshow(frame_to_display)
  cv2.waitKey()
    This code is rarely enough, especially when you need to, for example:
  • Navigation through the video back and forth frame by frame.
  • The ability to record the output to a file.
  • Supporting sources other than a simple video file (frame folder, stream, remote storage, etc.)
    • The most annoying thing: lack of interactivity.

    Using this kind of code, The visualization is created before rendering and cannot change once displayed.
    And, while this is okay for simple algorithms, for the more complex ones, there is just way too much information needed for each frame.
    And without the ability to decide, on the fly, what you want to display, you find yourself running the same video again and again, each time with different visualization parameters.

    Enter CV VideoPlayer

    CV VideoPlayer was born from the need for a simple customizable solution for interactively rendering videos and frames.
    It allows any number of overlays, sidebars, or any other frame edits, each of which can be easily switched on and off by the user during run time. let’s see an example of how this is done:

    Installation 

     
     pip install cvvideoplayer

    Playing vanilla video

    We can then import the video player and run an unedited video with the following code: 

     
from cvvideoplayer import create_video_player
VIDEO_OR_FRAME_FOLDER_PATH = ""
video_player = create_video_player(video_source=VIDEO_OR_FRAME_FOLDER_PATH)
video_player.run()

    This will open the video player and allow you to play it with the spacebar or using the arrows, it will also add some default built-in frame-edit-callbacks which we will elaborate on in the following section.

    Editing the frames to add visualization

    To add custom-built visualization to the video we can use the frame_edit_callbacks argument of the create_video_player constructor function like so: 

     
  from cvvideoplayer import VideoPlayer

VIDEO_OR_FRAME_FOLDER_PATH = ""

video_player = create_video_player(
video_source=VIDEO_OR_FRAME_FOLDER_PATH,
frame_edit_callbacks=[
        FitFrameToScreen(),
        FrameInfoOverlay(),
        KeyMapOverlay(),
    ]
)
video_player.run()

    Python Interview Questions

    1. Is Python a compiled language or an interpreted language?
      Ans Python is primarily an interpreted language, which means that Python code is executed line by line by an interpreter.
      However, there is more complexity to this, as Python uses a combination of interpreting and compiling.
    * Do you know What is Interpreted and compiled language?
    1. Interpreted Language:
      • Immediate Execution: Python code is executed directly by the interpreter without the need for a separate compilation step. This allows for quick testing and debugging
        Flexibility: Python supports dynamic typing and other runtime features that are easier to implement in an interpreted language.
    2. Compilation in Python:
      Bytecode Compilation: Before execution, Python code (.py files) is first compiled to an intermediate bytecode (.pyc files). This bytecode is a low-level representation that the Python interpreter can execute.️
      CPython Interpreter: The standard implementation of Python (C Python) interprets this bytecode using a virtual machine.
      Just-In-Time Compilation: Some implementations of Python, like PyP use JIT compilation to improve performance by compiling bytecode to machine code during execution.
    ➡️ The combination allows Python to be both powerful and user-friendly language

    Python's Sequence Protocol

    In Python, protocols are like informal interfaces—a collection of special methods that, when implemented, allow user-created objects to exhibit specific behaviors.
    The Sequence Protocol is a fantastic example. By implementing just a couple of special methods —__getitem__ and __len__— your custom objects can behave like standard Python sequences!

    Why is this so powerful?

    It means your objects can:
    • Iterate effortlessly using loops.
    • Support slicing, making subsets of data easy to work with.
    • Play nicely with Python's standard library, enabling reuse of Python's rich ecosystem without extra code.
    In the example, a Users class implements the sequence protocol. By defining __getitem__ and __len__, it transforms into a sequence-like container for user names. This lets you iterate, slice, and even apply functions like reversed() seamlessly—no extra boilerplate needed.

    Python’s sequence protocol is also the foundation for many built-in types like list, tuple, and str. By tapping into it, you’re leveraging the same underlying behavior that makes Python so intuitive!

    Web App with Flask

    Building Your First Web App with Flask
    Flask is often the gateway for many beginners into the world of web applications due to its simplicity and flexibility. If you’re new to web development, Flask provides you with the tools needed to build a web app quickly without overwhelming you with complexity. Let’s walk through what Flask is and why it’s a great choice for your first web application project.
    What is Flask?
    Flask is a lightweight WSGI (Web Server Gateway Interface) web application framework. It's designed to make getting started very easy and quick, with the ability to scale up to complex applications. It offers suggestions, but doesn't enforce any dependencies or project layout. This flexibility is one of its core strengths, making it particularly friendly for beginners.
    Why Choose Flask?
    1. Simplicity: Flask’s simple and readable codebase is ideal for beginners.
    2. Flexibility: Unlike other frameworks, Flask lets you use the components you need without requiring you to use a particular toolset.
    3. Extensibility: Flask can be extended with numerous extensions available to add features as if they were implemented in Flask itself.

    Creating a Basic Flask App
    Building a basic web application with Flask involves a few simple steps:
    1. Setup: Install Flask using pip, Python’s package installer.
    2. Create an App: Write a Python script that imports Flask and creates an instance of the Flask class. This instance becomes the WSGI application.
    3. Define Routes: A route is the URL where your application provides a web page. You define a route in Flask by writing a Python function and then using the Flask `route` decorator to associate the function with a URL.
    4. Run the App: Finally, run your application by calling `app.run()` in your script. This command starts a local development server that you can access through your web browser.

    Flask’s minimalistic approach doesn’t just ease your learning curve but also provides a deep sense of understanding and control over your application's structure and behavior. You get to learn the intricacies of web development without any components hidden away from you.
    By starting with Flask, you build a strong foundation in web application development, preparing you to tackle more complex frameworks and projects as you grow.
